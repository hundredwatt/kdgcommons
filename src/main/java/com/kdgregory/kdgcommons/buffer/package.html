Classes for working with java.nio.ByteBuffers and their close kin.

These classes were originally developed to work with mapped files that were over 2 gigabytes in size (larger than a single java.nio.MappedByteBuffer can access).

{@link com.kdgregory.kdgcommons.buffer.BufferFacade} is the primary abstraction: it allows indexed ("absolute") access to a buffer using long indexes, versus the int indexes of java.nio.ByteBuffer). It also provies bulk get operations that take/return a byte[]. And it gets rid of relative operations (they weren't that useful in my original applications).

{@link com.kdgregory.kdgcommons.buffer.MappedFileBuffer} memory-maps files, using a series of overlapping java.nio.MappedByteBuffers. By default, the overlap is 1 GB, allowing access to up to 2 GB of sequential data.

In addition to these classes, there is {@link com.kdgregory.kdgcommons.buffer.BufferUtil}, which provides utility methods for working with java.nio.ByteBuffer instances, and InputStream/OutputStream implementations that read/write data in a ByteBuffer.
